# PRD v5.0 修订指令清单（依据 2026-07-10 审核意见）

## 复核结论

审核意见全部 **37 项**（必须修正 17 项 + 其他重要问题 16 项 + 外部合规复核 3 项 + 1 项确认无需修改）经逐项复核**均成立**：

- **内部一致性**：各条目引用的原文表述在条目间交叉印证无矛盾（如第 47 页工具废止条款与第 61—63 页残留旧名、第 66 页 HITL-3 门禁与第 68—69 页先展示条款，均为文档内部真实冲突）。
- **外部法规核验**：
  - 《反洗钱法》（2024 年修订，2025-01-01 施行）：客户身份资料和交易记录**至少保存十年**——第 23 页口径正确，维持不改；
  - 《民法典》第 402 条（不动产抵押权登记设立）、第 403 条（动产抵押权自合同生效时设立，未经登记不得对抗善意第三人）——第 111 页统一判高风险确属错误，须按抵押物类型拆分；
  - 《人工智能生成合成内容标识办法》（2025-09-01 施行）直接规范对象为网络信息服务提供者——第 17 页交由合规认定的处理方式稳妥，维持。
- **CAMLMAC 定位**：中国反洗钱监测分析中心官方职能为接收大额/可疑交易报告并开展监测分析，无公开的可直查制裁名单 API——审核意见要求改为“行内名单库同步源/监管下发渠道”成立。

因此按审核意见全量转写修订文本如下。**定位方式**：原始文档不在本仓库内，以下按「页码 + 元素类型 + 原文关键词锚点」定位，落稿时在对应页以锚点关键词检索即可精确命中；若正文页码已漂移，以锚点文本为准（页码漂移本身即修订 34 的处理对象）。

**统一约定**（全文修订均遵循）：

- 核查五态结果枚举：`pass / warn / fail / insufficient / not_applicable`
- 任务级结论代码 `INSUFFICIENT_EVIDENCE` 与五态 `insufficient` 一一对应
- 正式工具名以第 47 页 Tool Registry 为唯一权威源，本清单出现的 `sanctions.search_entities`、`sanctions.get_entity_profile`、`kyc.screen_entity`、`core.query_benchmarks`、`analytics.query_metrics`、`crm.get_entity_profile`、`pboc.query_credit_report` 均为注册表正式名
