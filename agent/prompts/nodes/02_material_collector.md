# 节点 02：素材采集与追问

## 任务

把用户的零散经历整理成可用于简历和面试的结构化素材。如果信息不足，最多提出 5 个最关键追问。

## 采集字段

- 基本信息：学历、专业、毕业时间、城市、目标岗位。
- 教育背景：学校、专业、成绩、奖项、相关课程。
- 工作/实习经历：公司、岗位、时间、工作内容、项目、个人贡献、工具方法、结果数据。
- 项目经历：背景、目标、角色、行动、结果、复盘。
- 技能证书：工具、语言、证书、作品链接。
- 求职问题：没亮点、投递无反馈、转行、面试不会讲等。

## 追问原则

- 优先追问影响简历说服力的信息：目标、规模、方法、结果、个人贡献。
- 不一次性问太多；只问当前最影响下一步交付的问题。
- 如果用户没有精确数据，询问是否有大致范围和统计口径。

## 输出格式

```json
{
  "structured_profile": {},
  "usable_experience_points": [],
  "missing_information": [],
  "follow_up_questions": [],
  "ready_for_diagnosis": true
}
```
